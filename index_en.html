National School Survey / Data Consolidation

At the beginning of the project, I gathered databases from communities that responded to the letter sent by the Ministry of National Education and the Ministry of Ecological Transition, inviting them to send a list of schools in need of priority renovation according to three main criteria:

  • Energy consumption
  • Dilapidation
  • Pedagogical evolution affecting the building

As part of the National School Survey Project, carried out since November 2023 with the Ministry of National Education and Youth, I had the opportunity to develop my skills in data science and project management. This ambitious project aimed to survey the renovation efforts of school buildings across France. As the principal responsible for the project within the Bâti Scolaire team, I worked autonomously while receiving occasional help from my colleagues.

Context and project challenges: Our mission was to provide a comprehensive analysis to guide decisions related to sustainable educational infrastructure. This involved collecting precise information on the number of renovation projects and associated investments.

Objectives and constraints: A major constraint was the diversity of file formats (PDF, Word) and sources (rectorates, prefectures), requiring rigorous data standardization.

Approach: To overcome the difficulties related to managing data from different formats and sources, I learned to automate the integration of PDF and Word files into Excel. I developed techniques to insert and standardize data, which allowed me to improve my Excel skills. The approach included consolidating data from various sources into a centralized database and creating an intuitive dashboard to visualize the extent of renovations and facilitate strategic analyses for the ministry.

Results: This project resulted in the creation of a complete centralized database with detailed information by department and academy. An interactive dashboard was developed to show the needs, especially economic, of schools. This outcome highlighted the importance of sustainable infrastructure in educational institutions and established benchmarks for strategic advancements.

Skills developed and challenges encountered: Throughout this project, I developed and strengthened several key skills: data consolidation and management, project management, autonomy, adaptability. One major challenge was managing data from different file formats, especially converting from PDF to Excel. To overcome this, I learned to automate data integration, thus improving my Excel skills to effectively standardize formats.

Automated Web Data Collection

Context:

Our two-person team deployed advanced strategies in data extraction, API usage, and web scraping to enrich the SIRENE database.

Objectives and constraints:

Enrich the SIRENE database with geographical and additional information, facilitating geospatial analyses. A major constraint was managing large amounts of data, necessitating efficient processing methods. Additionally, we had to manage API rate limitations.

Approach:

To overcome difficulties related to managing data from different formats and sources, we used Python and Pandas for data extraction and manipulation, integrated the Address API to retrieve geographical coordinates, and used web scraping on Google Maps to locate each establishment. Managing very large files required us to work in chunks of 1000 lines to execute the code correctly. Additionally, we handled API rate limitations by introducing a 2-second delay to overcome these restrictions.

For those unfamiliar with these terms:

  • API (Application Programming Interface): A set of rules and tools allowing software applications to communicate with each other. In this project, it was used to retrieve geographical data.
  • Web Scraping: A technique used to extract large amounts of data from websites. In this project, it was used to collect location data from Google Maps.
  • Pandas: A Python library used for data manipulation and analysis. It provides data structures and functions needed to work efficiently with structured data.

Results:

This project successfully linked geographical coordinates to most establishments and enriched the database with additional information. An interactive dashboard was developed to show the needs, especially geospatial, of the establishments.

Skills developed:

Throughout this project, I enhanced my skills in data manipulation, API usage, and web scraping techniques. I also learned to manage large files and overcome technical limitations related to APIs.

Wind Energy Production Forecasting / Time Series Analysis

Context: At the US Energy Information Administration (EIA), our task was to explore time series related to electricity production from 2001 to 2022, focusing on wind energy production. This project was carried out as part of a SAE at IUT Paris Rives de Seine.

Objectives and Constraints: The main objective was to provide a thorough analysis of trends and forecasts for this essential component of the energy landscape.

Approach: We had to organize and manage our time effectively as we had limited hours. Initially, my colleague and I struggled to coordinate, but we overcame these difficulties towards the end. Additionally, using the R language required meticulous attention to detail and result reliability. It was crucial to carefully verify each line of code to ensure the accuracy of the results obtained.

Results: Through this project, we strengthened our skills in data analysis and time series forecasting. Applying techniques such as time series decomposition and forecasting models like Holt-Winters and ARMA highlighted hidden dynamics in wind energy production data. Our analyses provided precise forecasts for 2022 and 2023, revealing the characteristics and trends of wind energy production.

Skills: I developed project management skills, time series analysis, and programming with R.

Location: 143 Av. de Versailles, 75016 Paris - IUT Paris - Rives de Seine
